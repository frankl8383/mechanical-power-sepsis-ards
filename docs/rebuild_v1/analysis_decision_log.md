## 1. Rules for this log

1. Every material design, extraction, preprocessing, model, reporting, or interpretation change receives a unique ID.
2. Entries are append-only. A superseded decision is retained and points to the replacement ID.
3. A correction before rebuilt outcome access may be confirmatory if based only on schema, metadata, unit, timing, missingness, or reproducible code QC.
4. A change after rebuilt outcome access is exploratory unless it repairs an outcome-independent coding defect; the frozen result remains reported.
5. Any change to a value represented in code/R/rebuild_v1/00_config.R requires both a configuration version increment and a log entry.
6. “待核验” is a blocking status, not an invitation to choose a mapping from favorable results.

## 2. Decision status vocabulary

- **LOCKED:** may be implemented as written.
- **LOCKED-WITH-QC:** scientific rule is fixed; exact source mapping must pass outcome-blinded QC.
- **PENDING-BLOCKING:** implementation cannot proceed for that field until resolved.
- **SECONDARY:** prespecified but not primary.
- **EXPLORATORY:** cannot replace the primary analysis.
- **SUPERSEDED:** retained for provenance only.
